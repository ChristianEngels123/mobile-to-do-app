# Contributing to Mobile To-Do App

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing to this project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/mobile-to-do-app.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`

## Development Guidelines

### Code Style

- Use TypeScript for all new code
- Follow the existing code style
- Run `npm run lint` before committing
- Run `npm run format` to format your code

### Testing

- Write tests for new features
- Run `npm test` before submitting a PR
- Ensure test coverage is maintained

### Commit Messages

- Use clear, descriptive commit messages
- Start with a verb: "Add", "Fix", "Update", etc.
- Example: `Add todo completion toggle feature`

## Submitting Changes

1. Push your branch to your fork
2. Create a Pull Request against the main branch
3. Provide a clear description of your changes
4. Reference any related issues
5. Ensure all CI checks pass

## Questions or Issues?

Feel free to open an issue to discuss any questions or report bugs.
